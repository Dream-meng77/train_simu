# 05 — 上位机界面设计

## 5.1 界面整体布局

系统上位机界面采用经典的**管理信息系统布局**，分为三个主要区域：

```
┌──────────────────────────────────────────────────────────┐
│                    顶部状态栏 (AppHeader)                  │
│  [系统Logo] [系统状态LED] [实时统计]  [参考图按钮] [时钟]   │
├────────┬─────────────────────────────────────────────────┤
│ 左侧   │              主内容区 (main-area)                │
│ 导航栏  │ ┌─────────────────────────────────────────┐    │
│(Sidebar)│ │           站场平面图 (StationMap)          │    │
│         │ │           SVG 交互式站场图                │    │
│ ⇒进路   │ ├─────────────────────────────────────────┤    │
│ ⊞道岔   │ │          控制面板区 (动态切换)             │    │
│ ●信号   │ │   RoutePanel / TurnoutPanel /            │    │
│ ▶列车   │ │   SignalPanel / TrainPanel / LogPanel    │    │
│ ≡日志   │ └─────────────────────────────────────────┘    │
└────────┴─────────────────────────────────────────────────┘
```

### 5.1.1 布局CSS设计

- 整体采用 `flex` 布局，`height: 100vh` 充满视口
- 顶部栏：固定高度 52px
- 左侧导航栏：固定宽度 80px
- 主内容区：`flex: 1`，由站场图（固定高度）+ 控制面板（自适应填充）组成

## 5.4 站场平面图设计

组件：`StationMap.vue`（441行SVG交互图）

### 5.4.1 图形元素层次

```
第1层: 白色背景 rect
第2层: 棕黄色轨道线（静态 .rail 样式）
第3层: 道岔分支轨道（根据道岔位置动态接通/断开）
第4层: 区段彩色高亮覆盖线（透明叠加，状态变色）
第5层: 轨道设备标注
   - 绝缘节（青色竖向小矩形）
   - 轨道电路（双青色实心圆）
   - 安全挡车器（⊢型）
   - 坡度标注（6‰）
第6层: 道岔圆点（深灰外圈 + 状态色内点 + 编号/定反位文字）
第7层: 信号机（粉色方块 + 圆形灯头 + 编号文字）
第8层: PZA标注文字
```

### 5.4.2 区段颜色映射

| 区段状态 | 高亮层颜色 | 说明 |
|---------|----------|------|
| 空闲 | transparent | 不叠加，显示棕黄轨道本色 |
| 锁闭 | #fb8c00（橙色） | 进路已锁闭 |
| 占用 | #e53935（红色） | 列车占压 |
| 封锁 | transparent | 不叠加，与其他状态区别 |

### 5.4.3 道岔显示

| 组件 | 说明 |
|------|------|
| 外圈 | 根据锁定状态变色（灰=空闲/蓝=进路锁/紫=单锁/红=故障封锁） |
| 内点 | 根据道岔位置变色（绿=定位/橙=反位） |
| 编号 | 道岔编号（1/3/5/4/2） |
| 定/反位 | 当前位置文字标注（绿字"定"/橙字"反"） |
| 锁状态 | "锁"（蓝字）进路锁 / "S"（紫字）单锁 |

### 5.4.4 信号机显示

| 组件 | 说明 |
|------|------|
| 灯圆 | 根据灯色发光（绿/黄/红发光滤镜 + SVG filter 光晕） |
| 方块 | 粉色矩形（#e91e8c）信号机标识 |
| 编号 | 信号机编号文字 |
| 断丝 | 灯圆变灰色、无光晕 |

### 5.4.5 道岔分支轨道动态接通

```
道岔分支接通条件（computed）：
  arm3G  = 1#反位 AND 3#定位     (进站→3G)
  arm1G  = 1#反位 AND 3#反位     (进站→1G)
  arm4   = 4#反位                (出站3G汇合)
  arm2   = 2#反位                (出站1G汇合)
  armSF  = 5#反位                (1G→安全线)

接通：stroke = #c8960c（棕黄色，轨道本色）
断开：stroke = #dcdcdc（浅灰色）
```

### 5.4.6 交互操作

| 点击目标 | 触发行为 | 组件响应 |
|---------|---------|---------|
| 轨道区段（高亮层） | 切换占用/出清 | `occupySection()` / `clearSection()` |
| 道岔圆点 | 弹出道岔操作浮层 | `DevicePopup.vue (type='turnout')` |
| 信号机图标 | 弹出信号操作浮层 | `DevicePopup.vue (type='signal')` |
| 区段（弹出后） | 弹出区段状态浮层 | `DevicePopup.vue (type='section')` |
| 空白区域 | 关闭浮层 | `closePopup()` |

│ • 显示名称/类型     │ • 进路名称          │
│ • 显示路径/道岔     │ • 状态标签          │
│ • 状态高亮         │ • 区段列表          │
│                    │ • 建立时间          │
│ [锁闭] [开放]      │ • 信号机/道岔信息   │
│ [解锁] [取消]      │                    │
│                    │                    │
│ 取消倒计时进度条    │                    │
├────────────────────┴────────────────────┤
│  列车作业办理    │  调车作业办理          │
│  车次 + 类型选择  │  调车号 + 起止区段     │
│  [办理]          │  [办理]              │
└─────────────────────────────────────────┘
```

### 5.6.2 进路模板列表

每条进路显示：
- 名称（如"X→IIG（接车进正线）"）
- 类型标签（接车=蓝色 / 发车=绿色 / 调车=黄色）
- 路径区段序列（如 JXG → IIAG → IIG）
- 道岔要求（如 1#定）
- 状态指示（已激活进路高亮显示状态）

### 5.6.3 操作按钮逻辑

| 按钮 | 启用条件 | 执行函数 |
|------|---------|---------|
| 锁闭进路 | 选中进路 且 该进路未激活 | `establishRoute()` |
| 开放信号 | 选中进路 且 状态='locked' | `openRoute()` |
| 正常解锁 | 选中进路 且 状态='locked'\|'opened'\|'occupied' | `unlockRoute()` |
| 取消进路 | 选中进路 且 已激活 | `cancelRoute()`（先30s倒计时） |

### 5.6.4 取消倒计时设计

```
开始取消 → cancelCountdown = 30s
  → setInterval 每秒递减
  → 进度条动画同步（宽度从100%→0%）
  → 倒计时归零 → 自动调用 cancelRoute()
  → 可随时点击[立即解锁（人工）] → cancelRoute(force=true)
```

```

### 5.9.2 自动运行机制

```
启动自动运行:
  autoRunning = true
  setInterval(autoStep, autoInterval * 1000)

autoStep():
  for each 列车:
    找到包含列车当前区段的已开放/占用中进路
    若不在任何进路上 → 跳过
    若已到进路终点 → 跳过
    下一区段被占 → 跳过
    moveTrain(列车, 下一区段)  // 沿进路顺序推进
```

## 5.12 全局消息反馈

组件：`useNotify.js` composable

基于 Naive UI 的 `useMessage()` 封装，提供统一的全局消息条反馈：

| 方法 | 消息类型 | 颜色 | 用途 |
|------|---------|------|------|
| `notify(res)` | 自动判断 | 绿/红 | 根据 `{success, message}` 自动弹对应类型 |
| `ok(msg)` | success | 绿色 | 操作成功通知 |
| `err(msg)` | error | 红色 | 操作失败/被拒通知 |
| `warn(msg)` | warning | 橙色 | 警告通知 |
| `info(msg)` | info | 蓝色 | 一般信息通知 |
