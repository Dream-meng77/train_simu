# 06 — 仿真模块设计

## 6.1 仿真概述

本系统的仿真功能覆盖三类室外信号设备：信号机、道岔和轨道电路。仿真功能与联锁逻辑紧密结合，在确保安全性的前提下提供真实的设备状态变化模拟。

---

## 6.2 信号机仿真

### 6.2.1 信号开放仿真

信号机开放由联锁逻辑层驱动，在上位机界面实时反馈：

**界面表现**：
- 站场平面图信号灯圆：绿灯/黄灯 + SVG光晕滤镜效果
- 信号机面板：对应信号机卡片灯色同步变化
- 顶部状态栏：无直接变化

**实现机制**：
```
进路开放(openRoute) → 信号aspect = 'green'|'yellow'
  → 响应式传播至 StationMap.vue（sigColor/sigFilter计算属性）
  → SVG 圆形fill颜色 + filter 光晕效果变化
  → SignalPanel.vue 卡片同步更新
```

**信号灯色与滤镜**：

| 灯色 | fill颜色 | 光晕滤镜 | SVG filter |
|------|---------|---------|------------|
| 绿灯 | #43a047 | 绿色光晕 | `url(#gf-green)` - gaussianBlur(1.5) |
| 黄灯 | #fdd835 | 橙色光晕 | `url(#gf-orange)` - gaussianBlur(1.5) |
| 红灯 | #e53935 | 红色光晕 | `url(#gf-red)` - gaussianBlur(1.5) |
| 灭灯(故障) | #bdbdbd | 无光晕 | `none` |

### 6.2.2 信号关闭仿真

信号关闭由以下事件触发：

| 触发事件 | 机制 | 说明 |
|---------|------|------|
| 列车驶过内方 | `clearSection()` 检测到 path[0] 出清 | 信号自动复红 |
| 进路解锁 | `_doUnlock()` 显式设置 | 解锁时同步复红 |
| 进路兑现解锁 | `autoReleaseFulfilledRoutes()` | 列车到终点自动复红并解锁 |
| 取消进路 | `cancelRoute()` → `_doUnlock()` | 取消时复红 |

### 6.2.3 断丝故障仿真

**故障状态特征**：
- 信号灯圆变为灰色（#bdbdbd）
- 光晕效果消失
- 面板显示红色闪烁横幅"⚠ 信号机故障 — 断丝状态"
- 信号 aspect 强制设为 'red'
- 相关进路无法办理（openRoute 拒绝）

**故障设置与恢复**：

```
设置断丝(setSignalDefective(id, true)):
  → defective = true
  → aspect = 'red'
  → UI: 灯圆变灰 + 故障横幅

恢复(setSignalDefective(id, false)):
  → defective = false
  → aspect 保持 'red'（等待进路重新开放）
  → UI: 恢复正常灯色显示
```

**安全约束**：
- 断丝故障设置后，该信号机关联的所有进路在"开放信号"步骤被拒绝
- 不影响已开放的进路（但影响后续新办理的进路）

## 6.3 道岔仿真

### 6.3.1 道岔动作仿真

**单独操作道岔**（通过面板或点击浮层）：

```
操作触发 → operateTurnout(id, position)
  → 联锁检查（四重锁）
  → 设置道岔 position
  → 响应式更新界面:
    · 道岔内点变色（绿=定位, 橙=反位）
    · 道岔圆点下方文字变化（"定"/"反"）
    · 分支轨道实时接通/断开
```

**定位**：内点绿色(#43a047)，标注"定"（绿色字）
**反位**：内点橙色(#fb8c00)，标注"反"（橙色字）

### 6.3.2 道岔位置状态仿真

道岔外圈颜色根据锁定状态变化：

| 状态 | 外圈 stroke 颜色 |
|------|-----------------|
| 空闲 | #616161（深灰） |
| 进路锁闭 | #1e88e5（蓝色） |
| 单锁 | #8e24aa（紫色） |
| 故障/封锁 | #e53935（红色） |

### 6.3.3 故障状态仿真

**故障设置**：

```
setTurnoutDefective(id, true):
  → defective = true
  → blocked = true（自动封锁）
  → UI: 外圈变红，面板显示"故障"标签
  → 该道岔所有操作被拒绝
  → 相关进路办理时拒绝该道岔
```

**故障恢复**：

```
setTurnoutDefective(id, false):
  → defective = false
  → blocked = false（自动解封）
  → UI: 恢复正常状态显示
  → 道岔恢复可操作
```

### 6.3.4 分支轨道动态接通仿真

这是道岔仿真的核心视觉效果。站场图上道岔分支轨道的颜色随道岔位置实时变化：

| 分支 | 接通条件 | 接通颜色 | 断开颜色 |
|------|---------|---------|---------|
| JXG→3G | 1#反位 AND 3#定 | #c8960c（棕黄） | #dcdcdc（浅灰） |
| JXG→1G | 1#反位 AND 3#反 | #c8960c（棕黄） | #dcdcdc（浅灰） |
| 3G→IIBG | 4#反位 | #c8960c（棕黄） | #dcdcdc（浅灰） |
| 1G→IIBG | 2#反位 | #c8960c（棕黄） | #dcdcdc（浅灰） |
| 1G→SFXG | 5#反位 | #c8960c（棕黄） | #dcdcdc（浅灰） |

分支轨道颜色通过 Vue `computed` 属性实时计算，响应道岔 position 变化。

## 6.4 轨道电路仿真

### 6.4.1 占压仿真

**触发方式**：

| 方式 | 操作 | 说明 |
|------|------|------|
| 站场图点击 | 点击区段高亮线 | 直接切换占用/出清（受联锁门控约束） |
| 添加列车 | TrainPanel添加列车 | 自动占用指定区段 |
| 移动列车 | TrainPanel移动列车 | 出清原区段+占用新目标区段 |

**占压安全约束**：

```
占用区段(occupySection):
  区间(JXG/JSG): 允许自由占用 // 模拟区间来车
  站内区段: 检查是否有已开放/占用中的进路包含该区段
    → 若无开放进路 → 拒绝："无开放进路（信号红灯），不能占用"
```

**占压界面效果**：
- 站场图区段高亮变红色（#e53935）
- 面板区段状态显示"占用"
- 进路状态从 opened → occupied
- 联锁日志记录"轨道占用"

### 6.4.2 出清仿真

**触发方式**：

| 方式 | 操作 | 说明 |
|------|------|------|
| 站场图点击 | 点击已占用区段 | 切换为出清 |
| 移动列车 | 列车离开当前区段 | 自动出清 |
| 出清列车 | 删除列车 | 自动出清所在区段 |

**出清自动复红机制**：

```
出清区段(clearSection):
  for each 活动进路(status='opened'|'occupied'):
    if 该区段 === 进路.path[0]（内方第一区段）:
      防护信号机 aspect = 'red'
      记录日志 "信号自动复红"
```

**出清界面效果**：
- 站场图区段高亮恢复透明（显示棕黄轨道本色）
- 面板区段状态恢复"空闲"
- 信号机自动复红
- 若列车已到达进路终点且前段全部出清 → 触发进路兑现自动解锁

### 6.4.3 进路兑现自动解锁

这是轨道电路仿真的一个重要自动化逻辑：

```
autoReleaseFulfilledRoutes():
  触发条件: 每次 occupySection() / clearSection() 后自动调用

  检查每个 status='occupied' 的活动进路:
    last = path[路径最后一个区段]
    earlier = path[路径前段]
    if last被占用 AND earlier全部出清:
      → 进路兑现！
      → 解锁道岔/区段
      → 信号复红
      → 移除活动进路
      → 记录日志 "进路兑现，列车到达终点，进路自动解锁"
```

## 6.5 列车自动运行仿真

### 6.5.1 自动运行机制

组件：`TrainPanel.vue` 中的 `autoStep()` 函数

**启动**：
- 设置运行间隔（1~30秒）
- 点击"启动自动运行"

**每步执行逻辑**：

```
autoStep() [每个间隔触发]:
  for each 列车 in trains:
    步骤1: 找到包含列车当前区段且已开放/占用中的进路
    步骤2: 如果没有进路 → 跳过（等待用户办理进路）
    步骤3: 定位列车在进路path中的索引位置
    步骤4: 如果已到进路终点 → 跳过（等待后续进路）
    步骤5: 获取下一个区段（path[index+1]）
    步骤6: 如果下一区段被占 → 跳过（等待前车离开）
    步骤7: 调用 moveTrain() 移动列车到下一区段
    步骤8: 记录列车作业日志
```

**停止**：
- 点击"停止自动运行" → `clearInterval(autoTimer)`
- 组件卸载时（`onUnmounted`）→ 自动清除定时器

## 6.6 故障仿真汇总

### 6.6.1 故障类型与影响

| 故障类型 | 设备 | 触发操作 | 联锁影响 |
|---------|------|---------|---------|
| 道岔故障 | 任意道岔 | `setTurnoutDefective(id, true)` | 自动封锁，所有操作被拒，相关进路无法办理 |
| 信号断丝 | 任意信号机 | `setSignalDefective(id, true)` | 强制红灯，该信号机进路无法开放 |
| 区段封锁 | 任意区段 | `blocked = true` | 该区段不可被进路选排 |

### 6.6.2 故障恢复

| 故障 | 恢复操作 | 恢复后状态 |
|------|---------|----------|
| 道岔故障 | `setTurnoutDefective(id, false)` | defective=false，blocked=false，可正常操作 |
| 信号断丝 | `setSignalDefective(id, false)` | defective=false，aspect保持red，等待下一条进路开放 |
| 区段封锁 | `blocked = false` | 恢复为可参与进路选排 |

### 6.6.3 系统状态监控

顶部状态栏 `AppHeader.vue` 实时计算故障数：

```javascript
faultCount = SUM(
  turnouts.filter(t => t.defective || t.blocked).length,
  signals.filter(s => s.defective).length
)

if faultCount > 0:
  系统状态LED: 红色闪烁 "系统异常"
else:
  系统状态LED: 绿色常亮 "系统正常"
```

## 6.7 状态持久化

### 6.7.1 自动保存

所有仿真操作完成后自动调用 `railway.saveToStorage()`，将以下数据写入 `localStorage`：

- 所有轨道区段状态（occupied, locked, blocked）
- 所有道岔状态（position, locked, singleLocked, blocked, defective）
- 所有信号机状态（aspect, defective）
- 活动进路列表（含 status, 时间戳）
- 列车列表（含位置、方向）
- 最近 100 条操作日志

### 6.7.2 自动恢复

系统启动时调用 `railway.loadFromStorage()` 自动恢复上次运行状态。

### 6.7.3 系统复位

日志面板"系统全体复位"按钮：
- 所有设备恢复初始状态
- 清除所有活动进路和列车
- 清除 `localStorage`
- 记录"系统复位"日志

## 6.8 演示操作流程

### 6.8.1 接车进站演示

```
1. [进路面板] 选 X→IIG → 锁闭进路
   → JXG/IIAG/IIG 变橙(锁闭)，1#道岔自动转定位并锁闭
2. [进路面板] 点 开放信号
   → X进站信号自动变绿
3. [列车面板] 添加列车 K1234，区段 JXG
   → JXG 变红(占用)
4. 移动列车: JXG → IIAG → IIG
   或启动自动运行
   → 列车驶离JXG时 X自动复红
   → 列车到IIG时 进路自动兑现解锁
```

### 6.8.2 故障模拟演示

```
1. [道岔面板] 对1号道岔点"模拟故障"
   → 1号道岔外圈变红，自动封锁
   → 尝试办理 X→IIG 进路 → 被拒"1号道岔故障"
2. [信号机面板] 对X信号机点"模拟断丝"
   → X信号机灯圆变灰，显示断丝横幅
   → 尝试开放 X→IIG 信号 → 被拒"信号机故障"
3. 恢复正常 → 重新可办理进路
```

### 6.8.3 取消进路演示

```
1. 办理并锁闭一条进路
2. 点"取消进路" → 启动30秒倒计时
3. 进度条从100%→0% 动画
4. 可在倒计时期间点"立即解锁（人工）"
5. 倒计时结束或人工解锁 → 进路解锁，信号复红
```
